01 — Overview
What is VisionCloud?
VisionCloud is a full-stack cloud-native web application that lets users upload images and classify them using state-of-the-art computer vision models. The platform integrates five different AI architectures hosted on Hugging Face, stores images on Cloudinary CDN, persists results to Supabase PostgreSQL, and provides a rich analytics dashboard — all secured behind JWT-based user authentication.
Built as a B.Tech Computer Science capstone project with Cloud Computing specialization, VisionCloud demonstrates real-world cloud architecture patterns: microservices, serverless AI inference, CDN storage, managed databases, and continuous deployment.
02 — Features
Key Features
🤖
AI Image Classification
Upload any image and get instant AI-powered labels with real confidence scores from Hugging Face Vision models.
🏆
Multi-Model Benchmark
Run all 5 AI models in parallel on a single image and compare results, confidence and inference speed side by side.
🔐
User Authentication
Secure email/password registration and login via Supabase Auth with JWT session management.
📊
Analytics Dashboard
Visualize your scan history with scans-per-day charts, top label frequency bars and activity timeline.
🖼️
Cloud Image Storage
All images auto-uploaded to Cloudinary CDN with optimization, format conversion and permanent URLs.
🔍
Label Search
Search scan history by AI-detected labels — instantly filter images by what's in them, not just filename.
🌙
Dark / Light Mode
Full theme switching with localStorage persistence. Keyboard shortcut Ctrl+D to toggle instantly.
📱
Responsive Design
Fully responsive UI — desktop-first with clean mobile layout, touch-friendly interactions.

04 — AI Models
5 Vision AI Models
All models are hosted on Hugging Face Inference API and accessed via REST. Users can select any model or run all 5 in parallel via the Benchmark feature.

05 — API Reference
REST API Endpoints
Base URL: https://image-recognition-backend-93d6.onrender.com
GET / Health check — returns API status
POST /api/upload Upload image → Cloudinary → HF AI → Supabase
GET /api/history Fetch scan history (filtered by authenticated user)
POST /api/benchmark Run all 5 models in parallel on provided image URL
Example request body for /api/benchmark:
// POST /api/benchmark { "image_url": "https://res.cloudinary.com/your-cloud/image.jpg" } // Response { "success": true, "total_time": 4.2, "best_model": "google/vit-base-patch16-224", "fastest": "microsoft/resnet-50", "results": [ /* array of 5 model results */ ] }
06 — Architecture
System Architecture
VisionCloud uses a decoupled architecture with the frontend and backend deployed on separate platforms, communicating via REST API calls.
┌─────────────────────────────────────────────────────┐ │ USER BROWSER │ │ GitHub Pages (Frontend) │ └──────────────────────┬──────────────────────────────┘ │ REST API calls ┌──────────────────────▼──────────────────────────────┐ │ RENDER (Flask Backend) │ │ /api/upload /api/history /api/benchmark │ └───┬─────────────────┬──────────────────┬────────────┘ │ │ │ Cloudinary Hugging Face Supabase (Images) (5 AI Models) (DB + Auth)
